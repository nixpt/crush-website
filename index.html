What it is

🌐

Polyglot Embedding

Drop into another language mid-program with @python { } or @javascript { }. Python round-trips real values back into Crush today; JavaScript and Bash execute but don't yet return values.

🔑

Capability-Based

No ambient authority. A program gets io.print, fs.read, net.get only when you grant them — and nothing it wasn't handed. This is the core idea the language is built around.

⚙️

CASM & CVM1

Crush source compiles to CASM bytecode and runs on the CVM1 virtual machine — the same layer crushc, crush-run, and crush-repl are built on.

Where it's honestly at

Crush is alpha. Here's the real state, so you can decide what to try.

✅

Works today

The language, compiler, and VM run real programs: functions, structs, ranges (for i in 0..n), the capability system, and @python blocks with full value marshaling. cargo install gets you crushc / crush-run / crush-repl.

🚧

Rough edges

JavaScript and Bash blocks run but can't return values yet. There's no single crush umbrella command. The standard library is small. Multiple execution backends still have corners that disagree — we're building a differential harness to hunt those down.

⚠️

Known limitation

A polyglot block currently runs with the host process's full authority — it steps outside the capability sandbox. Don't run untrusted Crush yet. Bringing polyglot under the capability model is on the roadmap, and we'd rather say so than surprise you.
